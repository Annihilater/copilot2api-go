# copilot2api-go 使用教程

## 项目简介

`copilot2api-go` 是一个将 **GitHub Copilot** 转换为标准 AI API 代理服务的工具，基于 Go 编写。

**核心价值**：你只需要一个 GitHub Copilot 订阅，即可通过标准的 OpenAI / Anthropic 接口使用 Copilot 背后的大模型（GPT-4o、Claude Sonnet 等），并借助多账号管理和负载均衡大幅提升并发能力。

### 主要功能

| 功能               | 说明                                                   |
| ------------------ | ------------------------------------------------------ |
| OpenAI 兼容接口    | `/v1/chat/completions`、`/v1/models`、`/v1/embeddings` |
| Anthropic 兼容接口 | `/v1/messages`、`/v1/messages/count_tokens`            |
| 多账号管理         | 通过 Web 控制台管理多个 Copilot 账号                   |
| Pool 负载均衡      | 轮询或优先级策略分发请求                               |
| 模型 ID 映射       | 将 Copilot 内部 ID 映射为标准模型名称                  |
| 流式响应           | 完整支持 SSE 流式输出                                  |
| 管理员认证         | 密码保护的 Web 控制台                                  |
| 中英双语 UI        | 自动检测语言，支持手动切换                             |

### 默认端口

| 服务           | 端口    | 说明                 |
| -------------- | ------- | -------------------- |
| Web 控制台     | `37000` | 管理界面             |
| 代理 API       | `34141` | 对外提供 AI API 服务 |
| 前端开发服务器 | `35173` | 仅开发模式使用       |

## 2. 快速开始（推荐：源码运行）

### 方式 A：前台运行（开发/调试）

```bash
# 克隆项目
git clone https://github.com/your-username/copilot2api-go.git
cd copilot2api-go

# 直接运行（自动下载依赖）
go run . --verbose
```

启动后访问：

- Web 控制台：http://localhost:37000
- 代理 API：http://localhost:34141

### 方式 B：编译后运行

```bash
# 先编译前端（生成 web/dist/）
bash web/scripts/build.sh

# 编译 Go 后端
go build -o copilot-go .

# 运行
./copilot-go
```

### 方式 C：后台守护运行（生产推荐）

```bash
# 后台启动
bash internal/scripts/start.sh

# 查看状态
bash internal/scripts/status.sh

# 查看实时日志
bash internal/scripts/logs.sh

# 停止服务
bash internal/scripts/stop.sh
```

### 命令行参数

```bash
./copilot-go \
  --web-port=37000    # Web 控制台端口（默认 37000）
  --proxy-port=34141  # 代理 API 端口（默认 34141）
  --verbose           # 开启详细日志（默认关闭）
  --auto-start=true   # 启动时自动启用已保存的账号（默认开启）
```

> **端口说明**：默认端口使用了较偏僻的数字（37000 和 34141），以避免与常见服务冲突。如需修改，通过环境变量或参数指定：
>
> ```bash
> WEB_PORT=18080 PROXY_PORT=19000 bash internal/scripts/start.sh
> ```

## 3. Homebrew 安装（macOS 推荐）

macOS 用户可以通过 Homebrew 一键安装并用 `brew services` 管理服务生命周期（开机自启、后台常驻），无需手动管理进程。

### 安装

```bash
# 添加 tap（只需执行一次）
brew tap Annihilater/tap

# 安装
brew install copilot-go
```

### 服务管理

```bash
# 启动服务（设置为开机自启 + 立即启动）
brew services start copilot-go

# 停止服务
brew services stop copilot-go

# 重启服务
brew services restart copilot-go

# 查看所有 brew 服务状态
brew services list
```

启动后访问：

- Web 控制台：http://localhost:37000
- 代理 API：http://localhost:34141

### 日志位置

```bash
# 服务日志
tail -f $(brew --prefix)/var/log/copilot-go.log
```

### 修改端口

brew services 默认使用 `--web-port=37000 --proxy-port=34141`。如需修改，编辑 plist 文件：

```bash
# 查找 plist 路径
ls ~/Library/LaunchAgents/ | grep copilot

# 用文本编辑器打开并修改 ProgramArguments 中的端口参数
open ~/Library/LaunchAgents/homebrew.mxcl.copilot-go.plist

# 重新加载
brew services restart copilot-go
```

### 升级

```bash
brew upgrade copilot-go
brew services restart copilot-go
```

### 卸载

```bash
brew services stop copilot-go
brew uninstall copilot-go
```

## 4. Docker 部署

### 使用 Docker 直接运行

```bash
# 构建镜像
docker build -t copilot-go .

# 运行（持久化数据到 volume）
docker run -d \
  --name copilot-go \
  -p 37000:3000 \
  -p 34141:4141 \
  -v copilot-data:/root/.local/share/copilot-api \
  --restart unless-stopped \
  copilot-go
```

> **注意**：Dockerfile 内默认使用 3000/4141 端口，上面示例将宿主机的 37000/34141 映射到容器内。你也可以通过命令行参数修改容器内端口：
>
> ```bash
> docker run -d -p 37000:37000 -p 34141:34141 \
>   copilot-go --web-port=37000 --proxy-port=34141
> ```

### 使用 Docker Compose

创建 `docker-compose.yml`：

```yaml
services:
  copilot-go:
    build: .
    ports:
      - "37000:3000"
      - "34141:4141"
    volumes:
      - copilot-data:/root/.local/share/copilot-api
    restart: unless-stopped

volumes:
  copilot-data:
```

```bash
docker compose up -d
```

### 数据存储位置

所有数据持久化在 `~/.local/share/copilot-api/`（容器内）：

| 文件               | 内容             |
| ------------------ | ---------------- |
| `accounts.json`    | 账号列表与 Token |
| `admin.json`       | 管理员密码哈希   |
| `pool-config.json` | Pool 模式配置    |
| `model_map.json`   | 模型 ID 映射表   |

## 5. 添加 Copilot 账号

### 步骤一：首次访问，创建管理员账号

1. 打开浏览器，访问 `http://localhost:37000`
2. 首次使用会提示创建管理员账号
3. 设置管理员用户名和密码（请妥善保管）
4. 登录进入控制台

### 步骤二：通过 GitHub OAuth 添加账号

1. 点击控制台右上角的 **「添加账号」** 按钮
2. 选择 **「GitHub OAuth 设备流」** 方式
3. 系统会显示一个 **8 位验证码**（如 `ABCD-1234`）
4. 点击弹出的链接（或手动访问 `https://github.com/login/device`）
5. 在 GitHub 页面粘贴验证码并授权
6. 授权完成后，账号自动添加到控制台

### 步骤三：启动账号实例

1. 在账号列表中找到刚添加的账号
2. 点击 **「启动」** 按钮
3. 状态变为 **「运行中」** 即表示成功
4. 此时可以看到该账号的 **API Key**（格式：`sk-xxxx`）

## 6. 调用 API

### 获取 API Key

在 Web 控制台的账号详情页可以看到 API Key，格式为 `sk-` 开头的字符串。

### OpenAI 兼容接口

**对话补全（Chat Completions）**

```bash
curl http://localhost:34141/v1/chat/completions \
  -H "Authorization: Bearer sk-your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-4o",
    "messages": [
      {"role": "system", "content": "你是一个有帮助的助手"},
      {"role": "user", "content": "介绍一下你自己"}
    ]
  }'
```

**流式响应**

```bash
curl http://localhost:34141/v1/chat/completions \
  -H "Authorization: Bearer sk-your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-4o",
    "messages": [{"role": "user", "content": "写一首关于春天的诗"}],
    "stream": true
  }'
```

**查询可用模型**

```bash
curl http://localhost:34141/v1/models \
  -H "Authorization: Bearer sk-your-api-key"
```

**文本嵌入**

```bash
curl http://localhost:34141/v1/embeddings \
  -H "Authorization: Bearer sk-your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "text-embedding-ada-002",
    "input": "这段话需要向量化"
  }'
```

### Anthropic 兼容接口

**消息 API（Messages）**

```bash
curl http://localhost:34141/v1/messages \
  -H "x-api-key: sk-your-api-key" \
  -H "Content-Type: application/json" \
  -H "anthropic-version: 2023-06-01" \
  -d '{
    "model": "claude-sonnet-4",
    "max_tokens": 1024,
    "messages": [
      {"role": "user", "content": "你好，介绍一下自己"}
    ]
  }'
```

**Token 计数**

```bash
curl http://localhost:34141/v1/messages/count_tokens \
  -H "x-api-key: sk-your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "claude-sonnet-4",
    "messages": [{"role": "user", "content": "你好"}]
  }'
```

### 认证方式说明

代理服务支持两种认证头，按调用的 API 类型选用：

```bash
# OpenAI 风格（推荐用于 /v1/chat/completions）
Authorization: Bearer sk-your-api-key

# Anthropic 风格（推荐用于 /v1/messages）
x-api-key: sk-your-api-key
```

两种方式都能被正确识别，也可以混用。

## 7. Pool 模式（多账号负载均衡）

当你有多个 Copilot 账号时，Pool 模式可以将请求分发到多个账号，提升并发能力和稳定性。

### 开启 Pool 模式

1. 在 Web 控制台左侧菜单点击 **「Pool 配置」**
2. 开启 Pool 模式开关
3. 选择负载均衡策略：
   - **Round-Robin（轮询）**：请求均匀分配给所有运行中的账号
   - **Priority（优先级）**：优先使用高优先级账号，满载时才切换到低优先级
4. 保存配置，系统会生成一个 **Pool API Key**

### 使用 Pool Key 调用

Pool Key 的使用方式与普通 API Key 完全相同：

```bash
curl http://localhost:34141/v1/chat/completions \
  -H "Authorization: Bearer sk-pool-key-xxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-4o",
    "messages": [{"role": "user", "content": "你好"}]
  }'
```

> **提示**：Pool Key 会自动在所有**运行中**的账号间分发请求，单个账号下线不影响整体服务。

## 8. 模型 ID 映射

Copilot 内部使用非标准的模型 ID，映射功能可以将它们转换为你熟悉的标准名称。

### 配置映射

1. 在 Web 控制台点击 **「模型 ID 映射」**
2. 可以看到当前所有可用的 Copilot 模型 ID
3. 点击某个模型旁的 **「添加映射」**，填入你希望使用的标准名称

例如：

- Copilot 内部 ID：`claude-sonnet-4-20250514`
- 映射为：`claude-sonnet-4`

映射后，你就可以在请求中直接使用 `claude-sonnet-4` 这个名称，系统会自动转换。

### 映射规则

- `/v1/models` 接口返回映射后的标准 ID 列表
- 发送请求时，标准 ID 自动转换回 Copilot 内部 ID
- 映射关系持久化保存，重启后依然有效

## 9. 接入第三方工具

### Claude Code（推荐）

**方式一：环境变量**

```bash
ANTHROPIC_BASE_URL=http://localhost:34141 \
ANTHROPIC_API_KEY=sk-your-api-key \
claude
```

**方式二：项目配置文件**

在项目根目录创建 `.claude/settings.json`：

```json
{
  "env": {
    "ANTHROPIC_BASE_URL": "http://localhost:34141",
    "ANTHROPIC_AUTH_TOKEN": "sk-your-api-key",
    "ANTHROPIC_MODEL": "claude-sonnet-4",
    "ANTHROPIC_SMALL_FAST_MODEL": "gpt-4.1-mini"
  }
}
```

### OpenAI SDK（Python）

```python
from openai import OpenAI

client = OpenAI(
    api_key="sk-your-api-key",
    base_url="http://localhost:34141/v1"
)

response = client.chat.completions.create(
    model="gpt-4o",
    messages=[{"role": "user", "content": "你好"}]
)
print(response.choices[0].message.content)
```

### OpenAI SDK（Node.js）

```javascript
import OpenAI from "openai";

const client = new OpenAI({
  apiKey: "sk-your-api-key",
  baseURL: "http://localhost:34141/v1",
});

const response = await client.chat.completions.create({
  model: "gpt-4o",
  messages: [{ role: "user", content: "你好" }],
});
console.log(response.choices[0].message.content);
```

### Anthropic SDK（Python）

```python
import anthropic

client = anthropic.Anthropic(
    api_key="sk-your-api-key",
    base_url="http://localhost:34141"
)

message = client.messages.create(
    model="claude-sonnet-4",
    max_tokens=1024,
    messages=[{"role": "user", "content": "你好"}]
)
print(message.content[0].text)
```

### Cherry Studio / NextChat / Open WebUI

这类工具通常支持自定义 API 地址，配置如下：

| 字段     | 值                       |
| -------- | ------------------------ |
| API 地址 | `http://localhost:34141` |
| API Key  | `sk-your-api-key`        |
| 模型     | 从 `/v1/models` 接口获取 |

## 10. 脚本参考

项目提供了完整的管理脚本，分别位于 `internal/scripts/` 和 `web/scripts/`。

### 后端脚本（`internal/scripts/`）

| 脚本         | 用途                             |
| ------------ | -------------------------------- |
| `dev.sh`     | 前台启动，verbose 模式，适合调试 |
| `start.sh`   | 后台守护启动，适合生产环境       |
| `stop.sh`    | 停止后台运行的服务               |
| `restart.sh` | 重启服务（stop + start）         |
| `status.sh`  | 查看服务运行状态                 |
| `logs.sh`    | 实时查看日志（`tail -f`）        |

### 前端脚本（`web/scripts/`）

| 脚本         | 用途                               |
| ------------ | ---------------------------------- |
| `dev.sh`     | 前台启动 Vite 开发服务器（热更新） |
| `build.sh`   | 编译前端，产物输出到 `web/dist/`   |
| `start.sh`   | 后台启动 Vite 开发服务器           |
| `stop.sh`    | 停止后台 Vite 服务                 |
| `restart.sh` | 重启 Vite 服务                     |
| `status.sh`  | 查看 Vite 服务状态                 |
| `logs.sh`    | 查看 Vite 服务日志                 |

### 使用示例

```bash
# 开发流程：分别启动前后端
bash internal/scripts/dev.sh    # 终端 1：启动后端（前台）
bash web/scripts/dev.sh   # 终端 2：启动前端（前台，热更新）

# 生产部署：先编译前端，再后台启动后端
bash web/scripts/build.sh  # 编译前端
bash internal/scripts/start.sh   # 后台启动后端

# 运维操作
bash internal/scripts/status.sh  # 查看状态
bash internal/scripts/logs.sh    # 查看日志
bash internal/scripts/restart.sh # 重启服务
```

## 12. 常见问题

### Q：启动后访问 Web 控制台显示空白页？

**原因**：前端未编译，`web/dist/` 目录为空。

**解决方案**：

```bash
bash web/scripts/build.sh
bash internal/scripts/restart.sh
```

### Q：OAuth 授权后账号一直 pending？

**可能原因**：

1. GitHub 网络访问慢，等待 30 秒后重试
2. 已超过 GitHub device flow 的 15 分钟授权窗口，重新发起授权

### Q：调用 API 报 401 Unauthorized？

**排查步骤**：

1. 确认 API Key 是否正确（在 Web 控制台账号详情页查看）
2. 确认账号实例是否处于「运行中」状态
3. 使用 Pool Key 时，确认至少有一个账号在运行

### Q：响应速度很慢？

**可能原因**：Copilot Token 过期，正在自动刷新。Token 通常每小时刷新一次，刷新期间第一个请求会有 2-5 秒延迟，属于正常现象。

### Q：如何使用 HTTP 代理（科学上网）？

在 Web 控制台的「代理设置」中，填入你的代理地址，例如：

```
http://127.0.0.1:7890
```

保存后重启服务即可。

### Q：数据存在哪里，重装系统会丢失吗？

数据存储在 `~/.local/share/copilot-api/`，与程序分离。备份此目录即可保留所有账号和配置。

```bash
# 备份
cp -r ~/.local/share/copilot-api ~/copilot-api-backup

# 恢复
cp -r ~/copilot-api-backup ~/.local/share/copilot-api
```

### Q：能否修改默认端口？

可以，通过命令行参数或环境变量：

```bash
# 方式 1：命令行参数
./copilot-go --web-port=8080 --proxy-port=9090

# 方式 2：环境变量（使用脚本时）
WEB_PORT=8080 PROXY_PORT=9090 bash internal/scripts/start.sh
```

### Q：GitHub 提示账号触发了滥用检测？

减少并发请求数量，不要频繁轮询或发送大量请求。请遵守 [GitHub 可接受使用政策](https://docs.github.com/en/site-policy/acceptable-use-policies)。
